# CollabAgent MVP 完善 + Vue 前端开发计划

## 总体策略

- **后端零侵入**：不改动 `graph.py`、`agents/`、`tools/` 的现有逻辑（55 个测试全部保持通过），只在 `api.py` 层面新增 SSE 流式端点和静态文件托管。
- **前端独立开发**：放在 `collab-agent-mvp/web/`，开发用 Vite dev server + 代理，生产 `npm run build` 后由 FastAPI 托管。
- **单服务部署**：生产环境只需跑 `uvicorn api:app`，前端通过 `http://localhost:8000/` 直接访问，无跨域问题。

## 一、后端完善（`api.py`）

### 1. 新增 `POST /research/stream`（SSE 流式进度端点）

研究报告生成通常需要 30s~2min，原 `/research` 同步阻塞会让前端长时间卡 loading。新增 SSE 端点实时推送 Agent 工作进度。

**实现方式**：用 FastAPI `StreamingResponse` + 异步生成器，**不改动 agent 函数签名**，在端点层面分阶段编排并推送进度（agent 调用通过 `anyio.to_thread.run_sync` 放到线程池，避免阻塞 async 事件循环）。

**SSE 事件流设计**（`text/event-stream`）：
```
event: progress   data: {"stage":"research","message":"🔍 搜索研究员正在收集信息..."}
event: progress   data: {"stage":"research_done","message":"✅ 找到 5 条相关结果","count":5}
event: progress   data: {"stage":"writing","message":"✍️ 报告撰写员正在撰写报告..."}
event: progress   data: {"stage":"writing_done","message":"✅ 报告生成完成"}
event: complete   data: {"report":"# ...Markdown...","search_results":[...]}
event: error      data: {"message":"错误描述"}    （仅失败时）
```

**关键点**：
- 直接调用 `researcher_node(state)` → 推 `research_done`（含结果数）→ 调用 `writer_node(state)` → 推 `complete`。
- 不走 `compiled_graph`，避免在 graph 内部塞回调；原 `compiled_graph` 和 `/research` 完全不动。
- 异常时推送 `error` 事件并结束流。
- 复用现有 `ResearchState` 字段，行为与 `/research` 一致。

### 2. 静态文件托管（生产部署前端）

在 `api.py` 末尾用 `app.mount("/", StaticFiles(directory=..., html=True))` 托管 `web/dist`（构建产物）。挂载在 `/` 且在所有 API 路由之后注册，确保 `/research`、`/research/stream`、`/health` 优先匹配。

**兼容性处理**：用 `os.path.exists(dist)` 判断，开发期（未构建）不挂载，避免启动报错。

### 3. `/health` 保留现状

已有的健康检查端点（DD/Tavily/LLM Key 状态）前端直接复用，不改。

### 4. 后端测试

新增 `tests/test_stream.py`：mock `researcher_node` / `writer_node`，断言 SSE 端点依次产出 `progress(research)` → `progress(research_done)` → `progress(writing)` → `complete` 事件，以及错误路径推送 `error` 事件。保持"不依赖真实 API"的测试原则。

## 二、前端开发（`collab-agent-mvp/web/`，Vue 3 + Vite + Element Plus）

### 技术选型
| 用途 | 库 |
|---|---|
| 框架 | Vue 3（`<script setup>` 组合式 API）|
| 构建 | Vite 5 |
| UI | Element Plus（按需引入）|
| Markdown 渲染 | markdown-it |
| 代码高亮 | highlight.js |

### 目录结构
```
web/
├── package.json
├── vite.config.js          # dev: 代理 /research、/health → :8000；build 输出 dist
├── index.html
├── .gitignore              # node_modules/、dist/
└── src/
    ├── main.js             # 入口，注册 ElementPlus + 图标
    ├── App.vue             # 整体布局（顶部头 + 主体）
    ├── api/index.js        # 封装：fetchHealth()、researchStream(topic, onEvent)
    ├── composables/
    │   └── useResearch.js  # 研究状态机：idle/loading/done/error + 进度列表
    └── components/
        ├── HealthBadge.vue    # 顶部健康状态徽章（绿/橙/红 + tooltip）
        ├── TopicInput.vue     # 主题输入 + 示例 chips + 开始按钮
        ├── ProgressTimeline.vue  # el-timeline 实时显示 Agent 工作步骤
        ├── ReportView.vue     # Markdown 渲染区（含复制全文按钮）
        └── SourceList.vue     # 搜索来源卡片列表（标题/摘要/外链）
```

### 页面布局（标准版单页）
```
┌─────────────────────────────────────────────┐
│  CollabAgent · 多Agent研究报告     [● 服务正常] │  ← 头部 + HealthBadge
├─────────────────────────────────────────────┤
│  [ 研究主题输入框 __________________ ] [开始研究] │  ← TopicInput
│  示例: 2026年AI趋势 | 碳中和路径 | ...        │
├──────────────────────┬──────────────────────┤
│  进度时间线           │  [报告] [搜索来源]     │  ← 左：ProgressTimeline
│  ● 正在搜索信息...    │  ┌────────────────┐  │     右：Tab 切换 ReportView
│  ○ 找到 5 条结果      │  │ # Markdown 报告 │  │     / SourceList
│  ○ 正在撰写报告...    │  │  ...渲染内容... │  │
│                      │  └────────────────┘  │
└──────────────────────┴──────────────────────┘
```

### 关键交互
- **`researchStream`** 用浏览器原生 `EventSource`（POST 需用 `fetch` + `ReadableStream` 手动解析 SSE，因为 `EventSource` 只支持 GET——我会让 SSE 端点改为支持 `GET`，参数从 query 取 `?topic=`，更契合 `EventSource`，前端代码更简洁）。
  - **修正**：SSE 端点改为 `GET /research/stream?topic=xxx`，前端直接 `new EventSource('/research/stream?topic=...')`，监听 `progress` / `complete` / `error` 三个命名事件。
- **状态机**：`idle → loading（显示时间线）→ done（显示报告+来源）/ error（el-message 报错）`。
- **报告区**：markdown-it 渲染，顶部"复制全文"按钮用 `navigator.clipboard`。
- **来源卡片**：el-card 列表，标题 + 摘要 + "查看原文"外链（target=_blank）。
- **健康徽章**：页面加载时 fetch `/health`，根据 DD/Tavily/LLM Key 状态显示颜色与 tooltip。

## 三、集成与部署

### 开发模式（两个进程）
```bash
# 终端1：后端
cd collab-agent-mvp && uvicorn api:app --reload --port 8000

# 终端2：前端（Vite :5173，代理 /research /research/stream /health → :8000）
cd collab-agent-mvp/web && npm run dev
# 访问 http://localhost:5173
```
`vite.config.js` 配置 `server.proxy`，前端请求直接走相对路径。

### 生产模式（单服务）
```bash
cd collab-agent-mvp/web && npm run build     # 输出到 web/dist
cd .. && uvicorn api:app --port 8000          # 访问 http://localhost:8000
```
FastAPI 自动托管 `web/dist`，前端与 API 同源，零跨域。

### Docker
更新 `Dockerfile`：增加 Node 构建阶段，先把 `web/dist` 构建出来再拷进 Python 镜像，实现单容器部署。
